# Coworking Booking System

Веб-сервис для управления бронированием переговорных комнат.

Проект позволяет пользователям просматривать доступные помещения, проверять их доступность и бронировать их на нужное время (не более 6 часов подряд), а администраторам — управлять каталогом комнат и пользователями.

## Архитектура проекта

Проект реализован на базе монолитной трехуровневой клиент-серверной архитектуры (клиентское приложение, сервер приложения, сервер базы данных) с соблюдением методологии 12-факторного приложения (12-Factor App).

1. **Frontend (Клиентское приложение)**
   SPA-приложение на React. Позволяет пользователям просматривать карточки комнат, проверять их доступность и оформлять брони. Администраторам предоставляет панель для добавления и удаления переговорных. В production-режиме статические файлы раздаются через Nginx.

2. **Backend (Сервер приложения)**
   Единое монолитное приложение на FastAPI. Реализует всю бизнес-логику:
   - Аутентификация, выдача JWT-токенов и ролевая модель (RBAC).
   - CRUD-операции для каталога переговорных комнат.
   - Логика бронирования с проверкой коллизий и ограничений по времени.

3. **Database (Сервер базы данных)**
   Единая реляционная база данных PostgreSQL, хранящая пользователей, комнаты и бронирования со строгими внешними ключами.

4. **Gateway (API Gateway / Reverse Proxy)**
   Единая точка входа в приложение (порт 80). Обратный прокси-сервер на базе Nginx, который маршрутизирует запросы от клиента к бэкенду (/api/*) или отдает статику фронтенда (/).

## Используемые технологии

**Бэкенд:**
- Python 3.11
- FastAPI — веб-фреймворк для создания API.
- SQLAlchemy — ORM для работы с базами данных.
- PostgreSQL — реляционная база данных.
- Pytest — фреймворк для написания и запуска тестов.

**Фронтенд:**
- JavaScript / React
- Vite — инструмент сборки.
- Tailwind CSS — CSS фреймворк.

**Инфраструктура и DevOps:**
- Docker & Docker Compose — контейнеризация компонентов.
- Nginx — веб-сервер и обратный прокси.
- Yandex Cloud — облачная платформа для развертывания (Compute Cloud, Container Registry).
- GitHub Actions (CI/CD) — автоматическое тестирование, сборка образов и деплой на виртуальную машину.

## Запуск проекта локально (Разработка)

Для запуска проекта на вашем компьютере потребуется установленный Docker и Docker Compose.

1. Склонируйте репозиторий:
   ```bash
   git clone <URL_вашего_репозитория>
   cd KURS_BEK_6
   ```

2. Настройте переменные окружения:
   Скопируйте файл-пример конфигурации:
   ```bash
   cp .env.example .env
   ```

3. Соберите и запустите контейнеры:
   ```bash
   docker-compose up -d --build
   ```

4. Откройте приложение:
   Перейдите в браузере по адресу: http://localhost

Бэкенд покрыт Unit-тестами. Запустить тесты можно локально, установив зависимости:

```bash

pip install -r requirements.txt
PYTHONPATH=. python -m pytest tests
```
