# Coworking Booking System 🏢

Веб-сервис для управления бронированием переговорных комнат.

Проект позволяет пользователям просматривать доступные помещения, проверять их доступность и бронировать их на нужное время (не более 6 часов подряд), а администраторам — управлять каталогом комнат и пользователями.

## 🛠 Используемые технологии

**Бэкенд:**
- **Python 3.11**
- **FastAPI** — высокопроизводительный веб-фреймворк для создания API.
- **SQLAlchemy** — ORM для работы с базами данных.
- **PostgreSQL** — надежная реляционная база данных (используется изолированно для каждого микросервиса через драйвер `psycopg2`).
- **Pytest** — фреймворк для написания и запуска тестов.

**Фронтенд:**
- **JavaScript / React** — библиотека для создания пользовательских интерфейсов.
- **Vite** — быстрый инструмент сборки для современных веб-проектов.
- **Tailwind CSS** — utility-first CSS фреймворк для стилизации.

**Инфраструктура и DevOps:**
- **Docker & Docker Compose** — контейнеризация всех компонентов приложения в единую систему с использованием Multi-stage сборки (единый `Dockerfile` для всех микросервисов).
- **Nginx** — веб-сервер, выступающий в роли API Gateway (обратного прокси) для единой точки входа маршрутизации трафика.
- **GitHub Actions (CI/CD)** — автоматизация процессов проверки кода (Linting), запуска тестов и сборки Docker образов при пушах в репозиторий.

## 🧩 Архитектура микросервисов

Проект разбит на независимые сервисы, каждый из которых выполняет свою бизнес-логику и имеет собственную изолированную базу данных PostgreSQL.

1. **`auth_service` (Сервис аутентификации)**
   Отвечает за регистрацию пользователей, проверку учетных данных, выдачу и валидацию JWT-токенов, а также за ролевую модель (RBAC - Role-Based Access Control). Разделяет права доступа на обычных пользователей (`user`) и администраторов (`admin`).
2. **`room_service` (Сервис управления комнатами)**
   Обеспечивает CRUD-операции (создание, чтение, обновление, удаление) для каталога переговорных комнат. Хранит информацию о вместимости, описании, доступном оборудовании и фотографиях. Изменять каталог могут только администраторы.
3. **`booking_service` (Сервис бронирования)**
   Реализует основную бизнес-логику бронирования. Проверяет доступность комнат на запрашиваемые слоты времени, предотвращает пересечения (коллизии) бронирований и следит за соблюдением бизнес-правил (например, максимальная длительность одной брони — 6 часов).
4. **`frontend` (Пользовательский веб-интерфейс)**
   SPA-приложение на React. Позволяет пользователям просматривать карточки комнат, проверять их доступность и оформлять брони. Администраторам предоставляет панель для добавления новых переговорных.
5. **`gateway` (API Gateway)**
   Единая точка входа в приложение (порт 80). Обратный прокси-сервер на базе Nginx, который маршрутизирует запросы от клиента к нужному микросервису (`/api/auth/*`, `/api/rooms/*`, `/api/bookings/*`) или отдает статику и интерфейс React (`/`).

## 🚀 Запуск проекта локально

Для запуска проекта на вашем компьютере потребуется установленный Docker и Docker Compose.

1. **Склонируйте репозиторий:**
   ```bash
   git clone <URL_вашего_репозитория>
   cd KURS_BEK_6
   ```

2. **Настройте переменные окружения:**
   Скопируйте файл-пример конфигурации (если он еще не создан):
   ```bash
   cp .env.example .env
   ```
   *При необходимости вы можете изменить секреты и пароли в файле `.env`.*

3. **Соберите и запустите контейнеры:**
   ```bash
   docker-compose up -d --build
   ```

4. **Откройте приложение:**
   Перейдите в браузере по адресу: [http://localhost](http://localhost)

## Запуск тестов

Каждый сервис на Python покрыт Unit-тестами. Запустить тесты можно внутри контейнеров, пока проект запущен:

```bash
# Тесты сервиса аутентификации
docker-compose exec auth_service pytest

# Тесты сервиса комнат
docker-compose exec room_service pytest

# Тесты сервиса бронирования
docker-compose exec booking_service pytest
```
